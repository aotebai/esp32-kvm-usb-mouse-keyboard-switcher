# ESP32-S3 Flash Burn Addresses (烧录地址说明)
## 📋 核心烧录地址清单（必看）
| 固件文件名称 | 对应烧录地址 | 备注说明 |
|--------------|--------------|----------|
| `bootloader.bin` | `0x0` | 引导加载程序，ESP32-S3 启动必备 |
| `partition-table.bin` | `0x8000` | 分区表，定义固件、数据等分区的存储范围 |
| `ch9350_led_switch.bin` | `0x10000` | 项目核心固件（KVM 切换器功能实现） |
| `nvs_partition.bin`（可选） | `0x9000` | NVS 非易失性存储分区，用于保存设备配置参数 |

## ⚙️ 必选烧录参数配置
除了地址匹配，还需确保以下参数一致，否则会导致烧录失败或设备无法启动：
1.  **晶振频率**：`80MHz`（ESP32-S3 推荐默认值，不可改为 40MHz）
2.  **闪存模式**：`DIO`（兼容大多数 ESP32-S3 开发板闪存，避免使用 QIO 模式）
3.  **闪存大小**：`2MB`（若你的开发板闪存为 4MB/8MB，可选择对应大小，分区表需同步适配）
4.  **波特率**：`115200`（稳定传输，过高易出现传输错误，过低烧录耗时过长）

## 🚨 注意事项
1.  所有固件文件的烧录地址**不可随意修改**，必须与编译时的分区表配置一致，否则设备无法正常启动；
2.  若使用 ESP-IDF 重新编译固件，需确保 `partition.csv` 分区表文件中的地址与本文档一致；
3.  烧录时需按「先选择地址 → 再添加对应固件文件」的顺序操作，避免文件与地址不匹配；
4.  若烧录后提示「启动失败」「分区表错误」，优先检查地址配置和闪存大小参数是否正确。
